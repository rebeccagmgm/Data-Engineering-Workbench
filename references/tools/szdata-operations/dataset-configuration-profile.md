## 宽表是否走指标体系画像

所有数量都是 2026-07-04 本轮只读查询时点观察。

| 是否走指标体系 | 场景类型 | 本轮只读样本/查询时点观察 | 业务含义 | 配置差异 |
|---|---|---:|---|---|
| 走指标体系 | 通常无 `sceneType` | `szdata` usageType=3 观察到 5140 条；usageType=5 观察到 3 条 | 默认路径，适合可沉淀、可复用、模型加工/集市指标 | 不需要豁免场景；样本常有关联宽表和 Horae 任务 |
| 不走指标体系 | 场景1 | `szdata` usageType=3 观察到 4348 条 | 主数据、模型层基本属性 | 只需 `sceneType`；抽样 scene1 未观察到额外说明字段 |
| 不走指标体系 | 场景2 | `szdata` usageType=3 观察到 279 条 | 明细数据、订单数据 | 可不走指标体系，但仍需补齐宽表字段和分区信息 |
| 不走指标体系 | 场景3 | `szdata` usageType=3 观察到 10 条 | 紧急需求，开发流程长，临时绕行 | 需要整改时间；抽样记录观察到 `rectifierTime` |
| 不走指标体系 | 场景4 | `szdata` usageType=3 观察到 37 条 | 监管报送类 | 常与监管或报送语义相关 |
| 不走指标体系 | 场景5 | `szdata` usageType=3 观察到 63 条 | 指标时效性不足，跑数时间早于或不同于指标体系 | 适合中午、下午、清算后立即加工等时效诉求 |
| 不走指标体系 | 场景6 | `szdata` usageType=3 观察到 314 条 | 宽表所涉指标维度四维及以上 | 样本包含多维监管/明细类宽表 |
| 不走指标体系 | 场景7 | `szdata` usageType=3 观察到 3 条；`szdatatest` usageType=5 不走指标体系观察到 2 条 | 临时活动、短周期、不可复用场景 | 抽样 scene7 观察到场景说明和预期下线时间 |
| 实时宽表不走指标体系 | 测试多为场景7 | `szdata` usageType=5 本轮未观察到不走指标体系样本；`szdatatest` 观察到 2 条 | 推断：测试中更像回归/流程验证样本 | 推断：不能作为生产惯例；生产使用前应要求业务/治理确认 |

## 不走指标体系判断 Checklist

以下为基于样本和本地规则的 agent 判断策略，属于推断性操作建议，不替代平台校验或用户确认。

1. 先判断是否 `usageType=3` 或 `usageType=5`。只有宽表配置才进入“是否走指标体系”判断。
2. 默认选择走指标体系；只有命中 1-7 豁免场景才允许“不走指标体系”。
3. 优先识别场景1：字段主要是主数据、模型层基础属性，且无明显指标沉淀价值。
4. 若是明细/订单/监管/时效性不足/四维以上，再分别考虑场景2/4/5/6。
5. 若用户表达“赶时间”“先绕过”“流程来不及”，按场景3 处理，并要求整改时间。
6. 若是活动、临时比对、短周期、不可复用，按场景7 处理，并要求场景说明和预期下线时间。
7. 对实时宽表不走指标体系保持谨慎：生产本轮未观察到 usageType=5 不走指标体系样本；测试样本偏回归验证。
8. SQL 必须显式列字段，不要使用 `select *`；保存前必须重新获取输出列。
9. 输出字段备注/中文名要补齐；宽表数值字段要校验精度/长度。
10. 分区/日期字段重点检查：指标类常见 `busi_date`，测试宽表样本观察到 `busi_mon`，实时指标可见 `update_time`。
11. 宽表样本常有关联 `wideTableHoraeTaskId`；创建或编辑时不要只保存数据集，还要考虑后续宽表生成、预览、调度或升级链路。

## 后续 adaptor/skill 应沉淀的规则

| 规则 | 建议 |
|---|---|
| usageType 字典 | 固定识别为 1 持仓转换、2 离线指标生成、3 离线宽表配置、4 实时指标生成、5 实时宽表配置 |
| 宽表分支 | `usageType=3/5` 单独处理，不和普通 SQL 数据集共用保存/预览逻辑 |
| 指标体系默认值 | 默认走指标体系；`--indicator-system false` 必填 `sceneType` |
| sceneType 范围 | 仅沉淀 1-7；不要硬编码 scene8 |
| scene3 | 必填或强提示整改时间 |
| scene7 | 建议必填场景说明、预期下线时间 |
| 输出字段 | 保存前强制重新获取输出列；阻断 `select *` |
| 数值字段 | 宽表 `NUMBER` 字段校验精度/长度 |
| 状态字段 | 数据集列表状态不足；宽表状态应走 `widetable` / `widetable-detail` |
| 测试环境 | `szdatatest` 中 `测试`、`回归`、`勿删`、`xqz` 等样本只适合流程验证，不作为生产业务规律 |
| 查询输出 | 列表先分页，详情一次一个 ID；使用 SQL/字段预览参数，避免 dump 大 SQL、大 fieldList、大 JSON |

## 结论分层

| 类型 | 结论 |
|---|---|
| 生产样本事实 | 2026-07-04 本轮只读查询时点，`szdata` 中 usageType=3 离线宽表配置数量大，且走/不走指标体系数量接近；不走指标体系主要集中在场景1；宽表状态需通过 `widetable` / `widetable-detail` 读取。 |

| 推断 | 不走指标体系多用于主数据/模型属性、明细/监管/时效/复杂维度或临时场景；agent 应默认走指标体系，仅在命中豁免场景时选择不走；生产实时宽表不走指标体系不宜作为默认惯例。 |
