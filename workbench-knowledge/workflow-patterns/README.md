# Workflow Patterns

这里保存多个卡片、事件和证据门禁组合出来的常见工作流模式。

工作流模式不是固定流程图。它更像“遇到某类需求时，通常要警惕哪些未知、生成哪些卡、读回哪些证据”。

## 候选模式

- 新增数据推送链路分析。
- 采集批次新增和测试核验。
- 数据集新建与字段解析。
- 宽表预览、生成、调度和上线核验。
- DSP 接口上线前后核验。
- 调度权限排查。
- 目标表 DDL 不一致排查。
- 测试通过到生产读回。

## 模式模板

```markdown
# <模式名>

## 适用场景

## 常见输入

## 典型未知

## 常见卡片

## 关键证据

## 完成标准

## 常见误判

## 可晋升项
```

## 使用规则

- 每个模式必须说明“不适用场景”或常见反例。
- 模式只提供起点，不能把动态拆解变成固定菜单。
- 模式如果依赖具体 CLI 命令，应链接到能力契约而不是直接写死命令。
